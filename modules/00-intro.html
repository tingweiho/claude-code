A field guide by Ting-Wei

The Claude Code Playbook

How a senior data scientist actually runs the day-to-day: config, skills, automation, and memory — in one honest tour.

Hi — I'm Ting-Wei. Claude Code has quietly become the backbone of how I work: writing SQL, shipping code, drawing charts, drafting updates, and running the boring stuff while I'm asleep.

This page is a tour of that setup — not a manual, just the pieces that earned their place and a few lessons that cost me something to learn.

One caveat before we start: this is simply how I work — it may or may not be best practice. I try to follow it as consistently as I can and keep refining it by learning on my own, trial and error. So treat this as ideas and experience sharing, not a rulebook.

⌨️
Terminal or app — your choice

I personally like driving Claude Code from the terminal — it's where I live all day. But none of this is terminal-only: everything here works just as well in the Claude desktop and web apps. Use whatever fits your flow.
