記憶與自我精進：回饋的循環

一次性的更正如何變成永久的習慣——讓 Claude 從工具變成隊友的回饋循環。

一位真的記得住的同事

告訴一位好員工一次——「我們一律用 merge，絕不 rebase」——你就再也不必重複。記憶正是讓 Claude 成為那位員工，而不是每週一都要重新訓練的那位的關鍵。

memory/ 約 20 個單一事實檔案——每個只放一件事
MEMORY.md 索引：每則記憶一行，每個工作階段都會載入
feedback-viz-no-gridlines.md 我喜歡的工作方式——更正與已確認的做法
project-pro-plan-cutout.md 程式碼無法告訴你的、持續進行中的脈絡
reference-confluence-map.md 指向各種 ID、文件、儀表板的指標

約 20 個單一事實檔案。每個檔案一件事——好新增，也好淘汰。

一次更正如何變成永久

我第一次更正 Claude 時，它到下個工作階段就忘了。現在不會了——因為那次更正被寫進了像這樣的一個小檔案。

CODE 
---
name: feedback-viz-no-gridlines
description: No gridlines + centered titles by default
metadata:
  type: feedback
---
預設不要格線、標題置中；適用於
每一個繪圖函式庫。原因：更乾淨、墨水更少。
PLAIN ENGLISH

一個 frontmatter name——一個簡短的 slug，讓我能連結到這則確切的記憶。

一段 description——Claude 會掃描的那一行，用來判斷這則記憶此刻是否相關。

事實本身，再加上原因，這樣它才會帶著判斷力套用——而不是盲目照做。

💡
說一次，永遠套用

我只說過一次「拿掉格線」。現在每張圖一開始就是那樣——橫跨 Matplotlib、Plotly，任何東西都一樣。

隨時間愈收愈緊的循環

每一次更正都讓下一個工作階段更敏銳一點。這是一個循環——我用得愈多，這套設定就愈好。

1

我在對話中更正 Claude

→
2

它寫出一個單行的記憶檔案

→
3

下個工作階段，索引載入，而 description 與我的任務相符

→
4

行為改變了——不需要任何提醒

🧠
三層彼此強化

CLAUDE.md 說明我是誰，skills 說明如何執行某個流程，而 memory/ 存放對前兩者的更正。

一件事實該住在哪裡？

不是每件事都屬於記憶。每一種知識都有它的歸屬——而把它放進對的地方，正是這門手藝的一半。

👤

CLAUDE.md

穩定的身分與全域偏好——「精簡一點」、「SQL 用 snake_case」。永遠為真的事。

🔨

一個 skill

一個你會重複的流程——「如何 QA 一個 dbt model」、「如何排版一份給領導層的更新」。

📝

memory/

一個曾經讓你意外、且不該再意外的特定更正或事實。

📋
known_issues.md——一份持續累積的修正紀錄

約 16 則「這裡壞了、原因在此、修法在此」。

🛡️
regression-guard——一個安全防護 skill

強制執行基準 → 改動 → 重新驗證，這樣新功能就不會悄悄弄壞舊的路徑（一個迴歸）。

真實的常設指令

這些是實際存下來的學習成果——這套系統在運作中。每一則都曾是一個摩擦的瞬間，後來變成了永久的習慣。
