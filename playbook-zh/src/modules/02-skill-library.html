技能庫：隨需載入的專業知識

隨需載入的專業知識——一組隨傳隨到的專家，只在任務對得上時才現身。

我從不點名召喚某位專家

我從不輸入「用 SQL 技能」。我只描述我想要什麼，對的專業知識就會自己載入進來。一個 技能 就只是一個資料夾，裡面裝著一份 SKILL.md 檔案，以及一行 描述 來決定它何時啟動。

company-data-stack/ 一個技能 = 一個資料夾
SKILL.md 完整的指示，加上開頭那行觸發描述

技能庫一覽

十六個技能，依工作性質分成三個小隊。每個小隊有自己的顏色，貫穿整個課程。

🏢

公司營運

領域小隊。company-data-stack（資料倉儲＋ BI）、canvas-tracker（Slack 專案追蹤器）、scheduled-triggers 與 internal-comms——再加上 company-slack、company-confluence、gitlab-cli。

📊

產出與分析

data-analysis、data-viz、company-slides，以及 codebase-to-course——正是打造這個頁面的那個技能——還有 pdf 和 hex。

🧰

後設

prompt-authoring、skill-creator——做出技能的技能——還有 regression-guard。

16 個技能，依工作分組——而非按字母排序。

這些技能不是被點名召喚的——它們是憑意圖對上的。

描述本身就是觸發器

打開任何一份 SKILL.md，最上面幾行就把整個故事說完了。這裡是 frontmatter ——也就是 data-viz 技能的開頭那段。

CODE 
---
name: data-viz
description: Use whenever making a chart, plot, or
  visualization in a notebook. Trigger on "plot this",
  "visualize", "show the distribution", "compare segments".
---
PLAIN ENGLISH

每個技能都從一個名稱和一段描述開始。

名稱只是資料夾的標籤。

只有這段描述會一直待在 Claude 的記憶裡，直到被需要為止。

那些範例語句就是觸發線——只要你說出接近的話，完整的技能就會載入。

ℹ️
寫好一段描述，本身就是一門手藝

太籠統，技能就會一直觸發；太狹窄，它就永遠不觸發。甚至還有一個 prompt-authoring 技能，專門用來微調這件事——而且沒錯，一個 筆記本 就是資料分析師的工作畫布。

這些技能從哪裡來

這一切都不是從零打造的。整座技能庫是三種來源交織而成。

🛠️

從零自建

這些公司專屬的技能，封裝了任何下載都帶不來的內部知識——資料倉儲慣例、canvas 規則，以及公司內部的文件風格。

🍴

Fork 後重新品牌化

codebase-to-course，以及 company-slides 背後的 frontend-slides 引擎，都來自開源打造者 Zara Zhang （github.com/zarazhangrui）。我把它們 fork 下來，換上我自己的品牌與慣例。

📦

直接來自 Anthropic

skill-creator 和 pdf 是 Anthropic 自家的技能；ralph-loop 外掛 則從官方的 anthropics/claude-plugins-official marketplace 安裝。

💡
別從零開始打造

從一個 儲存庫 fork 一個好技能，重新品牌化，再把它彎成你自己的慣例——接著用 skill-creator 培養出你自己的技能。這是我們在最後會一起收攏的最佳實務線索之一。
