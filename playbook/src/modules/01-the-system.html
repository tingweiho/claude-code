The System

~/.claude/ as an operating layer — the files Claude Code reads before I type a word.

It already knows how I work

Every time I open Claude Code, it reads a handful of files before I type anything. Those files are why it already knows how I work.

This isn't a one-off prompt I paste in each morning. My config is a standing operating layer — a setup I version, improve, and carry from session to session.

💡
Aha!

The config is read top-to-bottom at startup. So order and brevity matter — the leaner the file, the more reliably Claude actually follows it.

The pegboard

Think of a workshop pegboard: every tool has a painted outline on the wall, so you see at a glance what's there and what's missing. ~/.claude/ is that pegboard — a fixed place for every kind of instruction.

~/.claude/ My whole operating layer — one folder I own
CLAUDE.md Who I am + how to talk to me + an index of my skills (48 lines)
skills/ 16 folders of on-demand expertise
settings.json Global guardrails, hooks, and data connections
settings.local.json The "yes, you may run this" allow-list
hooks/ Small scripts that fire on events
reference.md IDs, file paths, schedules I look up often
known_issues.md A running fixes log
*.sh statusline-command.sh, log_usage.sh, ci-watch.sh, auto_rebase.sh
memory/ Durable facts Claude has learned about me

A lean index file, a skills folder, two permission files, some hooks, and a memory folder. Treat it like a codebase, not a settings screen.

CLAUDE.md is a directory, not a manual

The file Claude reads first is short on purpose. It tells Claude how to behave, then points at where the real detail lives.

CODE 
## Communication
- Concise and direct — no filler, no trailing summaries
- Lead with the answer or action, not the reasoning

## Skills (load on demand)
- company-data-stack — Redshift / dbt / Looker / LookML
- canvas-tracker — Slack project tracker canvas ops
- data-analysis — rigor for diagnostic / exploratory work
PLAIN ENGLISH

First it tells Claude how to talk to me — short, answer-first.

Then it lists my skills with one line each on when they fire.

It doesn't explain the skills — it just points to them. The detail lives inside each skill folder.

📍
My whole CLAUDE.md is 48 lines.

It works as a table of contents that points outward, not a manual that holds everything. (We'll come back to why in the last module.)

Two permission files, one job each

Permissions are split across two files so the dangerous list stays short and the convenient list can grow freely.

🛡️
settings.json — global guardrails

A deny-list of destructive commands, plus the hooks and the data connections. A shared default that rarely changes.

✅
settings.local.json — the allow-list

An allow-list of ~322 specific commands I've pre-approved, so I'm not clicking "allow" all day.

Bash(rm -rf*) Never bulk-delete files
Bash(git push --force*) Never rewrite shared history
Bash(sudo*) Never run as administrator
🔎
Why split them?

Separating deny (global, rarely changes) from allow (local, grows over time) keeps the dangerous list short and auditable.

Connections — Claude plugs into my real tools

One standard, many tools

The data-stack connections run locally; the everyday apps are Claude's built-in connectors. Same idea either way — a standard socket, so adding a new tool doesn't mean re-teaching Claude how to talk to it.

Under the hood — one connection, redacted

Each connection is just a few lines of config. Here's the warehouse one — useful to recognize, but you never write it by hand often.

CODE 
"redshift": {
  "command": "uvx",
  "args": ["awslabs.redshift-mcp-server@latest"],
  "env": {
    "AWS_PROFILE": "<aws-account>_DataRedshift",
    "AWS_REGION": "us-west-2"
  }
}
PLAIN ENGLISH

Names a connection called redshift.

uvx downloads and runs a small helper program on the fly.

The settings say which account and region — the actual secrets live outside this file, never in plain text.

⚠️
A note on what you're seeing

Real configs hold live credentials — connection strings, client secrets, tokens. Everything shown in this course is redacted, and that discipline is itself part of the setup (more in the last module).
